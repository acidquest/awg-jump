# Future Integration Notes

Этот файл фиксирует только точки расширения после завершения `gateway-only` этапа. Реализации интеграции здесь нет.

## Подготовленные точки расширения

- `EntryNode.raw_conf` сохраняет исходный импорт, поэтому позже можно заменить импорт `.conf` на bundle без миграции структуры.
- `RoutingPolicy` и `GatewaySettings` отделены от runtime, поэтому будущий control client можно добавить отдельным слоем.
- diagnostics bundle уже агрегирует локальное состояние и может стать источником будущих telemetry snapshots.
- UI и backend не завязаны на tunnel API: текущие действия локальные и могут позже оборачиваться в control transport.

## Что не сделано специально

- нет клиента для control plane;
- нет удалённых команд `switch node`;
- нет передачи статистики наружу;
- нет синхронизации маршрутов, DNS или GeoIP с entry node.
